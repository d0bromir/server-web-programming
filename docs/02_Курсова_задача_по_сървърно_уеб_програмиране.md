Курсова задача – Сървърно уеб програмиране

# 1. Обща информация

Целта на курсовата задача е студентите да разработят минималистично уеб приложение с използване на сървърни технологии. Проектът трябва да демонстрира умения за работа с база данни, управление на потребителски сесии, реализиране на CRUD операции и използване на REST API. Достъпът до приложението се осъществява през браузър

# 2. Технически изисквания

• Реализацията може да бъде направена по избор на студента чрез PHP (версия 8+) или Java Spring Boot.

• База данни: PostgreSQL, SQLite, MySQL или MariaDB.

• Приложението трябва да има инсталационен скрипт, който да създава необходимите таблици, и да добавя акаунт по подразбиране.

• Валидация на входните данни.

• Защита срещу replay attack и cross-site scripting.

• При възникване на грешка по време на валидацията или обработката на данните, да се връща грешка с правилния HTTP Status Code

• Препоръчително е логиката за работа с база данни да е отделена от презентацията.  
• Конфигурацията на връзката към базата данни трябва да бъде отделена в конфигурационен файл.  
• Всички SQL заявки трябва да използват prepared statements.  
• Паролите трябва да се съхраняват само като криптографски хеш.

# 3. Основна функционалност

Приложението трябва да включва:  
• Публична част – списък със записи, достъпни за всички посетители.  
• Автентикация. Система за вход и изход (login/logout).  
• Управление на съдържание (CRUD): добавяне, редактиране и изтриване на записи от вписани потребители.  
• Запазване на избран или последно добавен елемент (използване на cookies или sessions).

• Търсене и филтриране на записи  
• При визуализация на данните да са сортирани и разделени на страници

# 5. REST API

Приложението трябва да предоставя собствено REST API.  
Минимални endpoints:  
GET /api/records – връща списък със записи във формат JSON.  
POST /api/records – създава нов запис.  
Заявките трябва да изискват токен за удостоверяване. Обикновено се ползва JWT token, но за целта на проекта, токенът може да е еднократно генериран достатъчно дълъг random string.

# 6. Външен REST API

Сайтът трябва да извиква поне един публичен REST API и да визуализира данните в уеб интерфейса. Резултатите трябва да се обработват от сървърната логика и да се показват в HTML.

# 7. Бонус задачи

За отлична оценка може да бъде реализирана една от следните допълнителни функционалности:  
• удостоверяване чрез OAuth/OpenID (Google, GitHub и др.)  
• интеграция с външен API, изискващ authentication  
• реализиране на собствен защитен API.
