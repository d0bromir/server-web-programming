CSRF – Cross-Site Request Forgery

Spring Security изисква CSRF токен за всяка POST заявка. Формата с th:action го получава автоматично. Формата без th:action няма токен → 403 Forbidden.
✅ Защитена форма (th:action)
❌ Без токен (обикновен action)
✅ Получено (заявката мина): 

<!-- Thymeleaf автоматично добавя: -->
<input type="hidden" name="_csrf" value="...токен..."/>

// CSRF е включена по подразбиране, защото:
//   - Браузърите автоматично изпращат cookies (вкл. session cookie) към всеки сайт.
//   - Злонамерен сайт може да накара браузъра на жертвата да изпрати заявка
//     към вашия сървър ─ и тя ще носи валидна session cookie.
//   - CSRF токенът е таен, известен само на вашата страница ─ фалшивият сайт
//     не може да го знае, затова заявката му ще бъде отхвърлена с 403.
//
// REST API-та обикновено са изключени, защото:
//   - Клиентите им (JS fetch, curl, мобилни app) не разчитат на cookies,
//     а на Authorization header (Bearer token, API key).
//   - Браузърите не добавят такъв header автоматично ─ няма риск от CSRF.
http.csrf(csrf -> csrf.ignoringRequestMatchers("/api/**"));
