# Примери – Сървърно уеб програмиране

Всеки пример e самостоятелен и може да се стартира директно.

## Структура

```
examples/
├── php/                    # PHP 8.3 примери
│   ├── 01-introduction/    # Въведение в сървърното уеб програмиране
│   ├── 02-http/            # HTTP протокол – заявки и отговори
│   ├── 03-mvc/             # MVC архитектурен модел
│   ├── 04-frameworks/      # Сървърни Framework-и – концепции
│   ├── 05-routing/         # Routing и обработка на заявки
│   ├── 06-database/        # Работа с бази данни и Repository
│   ├── 07-crud/            # CRUD операции и слоево разделение
│   ├── 08-sessions/        # Управление на сесии и cookies
│   ├── 09-authentication/  # Автентикация и авторизация
│   ├── 10-rest-api/        # RESTful уеб услуги
│   ├── 11-security/        # Сигурност (CSRF, XSS, валидация)

│
└── java/                   # Java 21 + Spring Boot 3.x примери
    ├── 01-introduction/
    ├── 02-http/
    ├── 03-mvc/
    ├── 04-frameworks/
    ├── 05-routing/
    ├── 06-database/
    ├── 07-crud/
    ├── 08-sessions/
    ├── 09-authentication/
    ├── 10-rest-api/
    ├── 11-security/

```

## Стартиране на PHP примери

```bash
# Влезте в директорията на примера
cd examples/php/01-introduction

# Стартирайте вградения PHP сървър
php -S localhost:8000

# Отворете http://localhost:8000 в браузъра
```

## Стартиране на Java примери

```bash
# Влезте в директорията на примера
cd examples/java/01-introduction

# Стартирайте с Maven
mvn spring-boot:run

# Отворете http://localhost:8080
```

> **Забележка за структурата на Java примерите:**  
> Всички класове във всеки пример са наmerени в **един единствен файл** (`Application.java`),
> за да се вижда ясно последователността и връзката между тях.  
> Това е **само за учебни цели** – в реален проект всеки клас се поставя в **отделен файл**
> според пакетната структура (например `controller/`, `service/`, `model/`).

## Курсова задача

Директориите `course-project/` съдържат референтна имплементация на тема **„Любими заведения"** (тема 7).  
Кодът може да се адаптира за всяка от 10-те теми от курсовата задача.

### Адаптиране за друга тема

1. Преименувайте `Venue` → вашата entity (Film, Food, Sport …)
2. Актуализирайте полетата в модела и таблицата
3. Сменете External API (вж. Приложение А в условието)
4. Обновете etiketi в изгледите (views)
